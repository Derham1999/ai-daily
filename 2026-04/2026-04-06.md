## **产品与功能更新**

1. **Gemma 4 登陆 iPhone，端侧 AI 进入实战阶段**
   Google 通过 [AI Edge Gallery](https://github.com/google-ai-edge/gallery) 将 Gemma 4 E2B/E4B 模型带上 iPhone 本地运行，用户实测 iPhone 16 Pro 达到约 **30 TPS**，同时伴随发热。评论中出现大量与 Qwen 3.5 在编码、推理能力上的对比，以及端侧推理在隐私、离线可用性上的独特价值讨论。Apple Foundation Models 和 NPU/Neural Engine 的角色也成为焦点。📱

2. **Gemma 4 26B 接入 Claude Code，本地 coding agent 工作流跑通**
   通过 [LM Studio headless CLI](https://newshacker.me/story?id=47651540) 和 Ollama 将 Gemma 4 26B 以 Anthropic-compatible endpoint 方式暴露给 Claude Code。社区发现默认 q4 量化下模型容易卡住，q8 更稳定；MoE 架构的显存优势和实际性能边界引发激烈争论。OpenCode、Cursor、Zed 等替代 coding harness 也被拿来横向对比。🔧

3. **OpenAI Codex 切换至 token 计费模式，Plus/Pro 用户即将受影响**
   Codex 将从按消息粗估额度改为按 [input/output/cached token 直接扣费](https://newshacker.me/story?id=47650726)，2x promo 同时结束。社区普遍解读为变相涨价，credits 虚拟单位被类比为"手游 gem pack"。更宏观的视角认为这标志着 AI 行业从增长优先转向回收成本，补贴期正式落幕。💰

4. **Claude HUD 插件发布，实时展示 Claude Code 工作状态**
   [Claude HUD](https://github.com/jarrodwatts/claude-hud) 是一个 Claude Code 插件，实时显示上下文用量、活跃工具、运行中的 agent 和 todo 进度，为开发者在编码工作流中提供更好的可视性。

## **行业展望与社会影响**

1. **TurboQuant 论文引发存储芯片市场恐慌，ML 社区：过度反应**
   TurboQuant 将 KV 缓存压缩至 3 bit，引发 [48 小时内数百亿美元市值蒸发](https://www.reddit.com/r/MachineLearning/comments/1sdb7ne/d_the_memory_chip_market_lost_tens_of_billions/)。但 ML 社区迅速指出：KV 缓存是推理内存，训练所需的激活值、梯度、优化器状态完全不受影响，而 HBM 需求主要来自训练。这已是 14 个月内第二次因 AI 效率论文恐慌抛售存储股。📉

2. **Marc Andreessen 连发多条：AGI 已来，技术岗位反弹至 3 年最高**
   Andreessen 在推特连续发文称 "AGI is already here — just not evenly distributed yet"，并引用 Business Insider 数据指出 [软件工程师岗位超过 67,000 个](https://www.businessinsider.com/ai-isnt-killing-software-coding-jobs-booming-trueup-2026-4)，为 3 年来最高，较 2023 年低点翻倍。他将 AI 取代论斥为"全是假的"。📊

3. **"AI 时代唯一护城河：难得到的 vs 难做到的"**
   一篇被广泛转发的文章将 AI 时代的护城河总结为：AI 正在让"难做到的"（写软件、搭产品）难度归零，但"难得到的"（千万用户、政府牌照、芯片工厂）需要真实时间积累。**时间才是元护城河**。🧱

4. **日本机器人填补劳动力缺口：不是抢工作，是维持服务运转**
   日本用机器人和自动化接手清洁、便利店等"没人愿意干"的岗位，背后是人口老龄化和低失业率的结构性缺工。评论对比美国 AI 更常替代知识工作者，指出不同国家的自动化方向截然不同。🤖

## **前沿研究**

1. **Fused MoE Dispatch in Pure Triton：纯 Triton 击败 CUDA 优化的 Megablocks**
   开发者在 A100 上用纯 Triton 实现的 [fused MoE dispatch kernel](https://github.com/bassrehab/triton-kernels) 在推理相关 batch size 下击败 Stanford 的 Megablocks（32 tokens 时 131%，128 tokens 时 124%），减少约 35% 内存流量，且在 AMD MI300X 上零代码修改通过测试。⚡

2. **Architecture Determines Optimization：从网络拓扑推导权重更新**
   一篇论文从第一性原理推导神经网络权重更新，不假设梯度下降或特定损失函数，证明梯度下降是"在误差修正约束下最小化权重变化"的自然结果。论文还从结构角度解释了灾难性遗忘的原因。🧠

## **社媒观点**

1. **"大多数人用 AI 的方式错了——把它当工具而非协作者"**
   Reddit 热帖指出，将 AI 视为"提示-提取-走人"的工具链存在天花板。真正有效的模式是把它当作协作者，在循环中不断精炼、挑战和构建。关键转变是从"用 AI"到"与 AI 一起思考"。🤝

2. **McKinsey 的 AI 谎言：35 年老数据库套了个自然语言接口**
   评论揭露 McKinsey 宣称的"25,000 AI 专家"实际上是把内部老数据库加了个 NL 接口就发了新闻稿。这不是第一次：90 年代的 ERP、2000 年的数字化转型、2010 年的大数据，每次都是同样的套路——贩卖焦虑、定位中间人、企业买单。🎭

3. **AutoAgent：Claude 评估 Claude 的自我进化 Agent**
   [AutoAgent](https://github.com/kevinrgu/autoagent) 开源了一个 Meta Agent 机制：用一个 Claude 实例调整另一个 Claude 的 harness（工具、系统提示等），反复测试改进直到冲顶基准。核心理念是"agent 差不是因为模型，而是因为 harness"。🔄

4. **"替代你的不是 AI，是隔壁部门学会了用 AI 的人"**
   一位战略岗位从业者分享：两年前写推荐方案交给产品团队，现在直接用 Claude 出原型。工程师开始做产品决策，产品经理做战略假设，战略人员直接写代码——**AI 正在模糊所有职能边界**。🔗
