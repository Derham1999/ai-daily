### 产品与功能更新

1. **Claude 正式打通 Microsoft 365 全家桶**
   [Claude](https://x.com/claudeai/status/2040086268562842097) 宣布 Microsoft 365 connectors 现已向所有 Claude 付费计划开放，用户可将 **Outlook 邮件**、**OneDrive 文件**和 **SharePoint 文档**直接接入对话。Ethan Mollick 随即评论称"**RAG 时代虽然短暂但极其激烈**"，RAG 不再是 Agent 获取上下文的主流范式——深度产品集成正在取而代之 📎

2. **Gemma 4 以小博大，性能惊艳**
   DeepMind 联合创始人 [Demis Hassabis](https://x.com/demishassabis/status/2040067244349063326) 发推展示 **Gemma 4** 的 benchmark 表现：在对数尺度坐标轴上，Gemma 4 的性能曲线显著超越参数量 **10 倍以上**的模型。开源小模型正以惊人速度逼近甚至超越闭源大模型的天花板 🚀

3. **Mintlify 用虚拟文件系统终结 RAG 痛点**
   文档平台 [Mintlify](https://x.com/dotey/status/2040157640442229153) 发布工程博客，介绍了名为 **ChromaFs** 的虚拟文件系统：让 AI 以为自己正在用 `grep`、`cat`、`ls` 浏览文件，实际每个命令都被拦截翻译为数据库查询。会话启动时间从 **46 秒降至 100 毫秒**，边际计算成本几乎为零。这是 RAG 范式转变的一个绝佳工程案例 ⚡

4. **Anthropic 揭示 Claude 内部"情绪"机制**
   Anthropic 发布最新[研究](https://x.com/dotey/status/2040159519607587304)，发现 Claude 内部存在类似"情绪"的机制，这些机制会切实影响模型行为，有时甚至会把输出"带歪"。研究者通过干预 Sonnet 4 的内部状态，展示了人格一致性维持的技术细节。这为理解 LLM 的内部表征提供了新视角 🧠

5. **NVIDIA 发布 NemoClaw，发力本地 Agent 生态**
   [NVIDIA AI Dev](https://x.com/NVIDIAAIDev/status/2040127202813956473) 在 DGX Spark Live 上演示了 **NemoClaw** 的快速上手流程。与此同时，Jeff Boudier 转发强调：本地 Agent 需要开源模型，私密 Agent 需要开源模型，Agent 工具链更需要开源模型——**这是开源模型的时代** 🟢

6. **Replit Agent 4 Buildathon 持续升温**
   [Replit](https://x.com/Replit/status/2040132609515413699) 公布第二周 Agent 4 Content Challenge 获胜者 Marcos Leal，其作品 [birdie.news](http://birdie.news) 是一个完全围绕用户个性化构建的每日新闻简报。同期 Replit 还预告了 [VibeCon.ai](http://vibecon.ai)，暗示将有重大发布 🎉

### 行业展望与社会影响

1. **OpenClaw 安全漏洞引发社区恐慌**
   一篇标题为"[如果你在运行 OpenClaw，你大概率上周已经被黑了](https://newshacker.me/story?id=47628608)"的帖子在 Hacker News 引发热议（评分 47）。核心争议在于 OpenClaw 默认配置将执行接口暴露公网，评论将其与 Claude Code、Codex 等 CLI harness 的安全模型进行对比。AI Agent 工具的安全性已不再是学术讨论，而是生产环境中的现实威胁 🔓

2. **AI 安全自动化呼吁：1 亿美元拨款**
   Apollo Research 成员在 [Alignment Forum](https://www.alignmentforum.org/posts/qdhyrN4uKwBAftmQx/there-should-be-usd100m-grants-to-automate-ai-safety) 发文呼吁资助方投入 **1 亿美元以上**用于 AI 安全工作的自动化。核心论点是：安全研究的规模化速度必须匹配 AI 能力的增长速度，否则将出现不可逆的安全缺口。这反映了安全社区对当前投入不足的深层焦虑 ⚠️

3. **Andrej Karpathy 发布 LLM Knowledge Base 架构图**
   [Karpathy](https://x.com/omarsar0/status/2040182254014460116) 分享了 **LLM Knowledge Base** 系统的完整架构图，引发广泛关注。用户只需将架构图喂给 Agent 即可搭建自己的知识库系统。这标志着从 RAG 到更复杂知识管理范式的过渡正在加速 📚

4. **Claude Agent 应用架构指南发布**
   Claude 团队技术成员 [Lance Martin](https://x.com/shao__meng/status/2040044601201844688) 发布长文探讨如何在智能、延迟与成本之间重构 **Agent 应用架构**。随着 Claude 模型持续变强，如何设计合理的 Agent 架构平衡三者关系，已成为所有 LLM Agent 开发者的核心课题 🏗️

### 开源TOP项目

1. **[f/prompts.chat](https://github.com/f/prompts.chat)** `HTML` ⭐ 157,186
   原 Awesome ChatGPT Prompts 的全面升级版，支持社区**分享、发现和收藏** Prompt，支持**私有化部署**，面向组织提供完整隐私保护

2. **[sherlock-project/sherlock](https://github.com/sherlock-project/sherlock)** `Python` ⭐ 78,519
   通过用户名在**数百个社交平台**上追踪账号，数字身份侦察利器，网络安全和 OSINT 社区的瑞士军刀 🔍

3. **[onyx-dot-app/onyx](https://github.com/onyx-dot-app/onyx)** `Python` ⭐ 23,284
   开源 AI 对话平台，兼容**所有主流 LLM**，提供高级功能和企业级特性，可作为 ChatGPT 的私有化替代方案 💬

4. **[siddharthvaddem/openscreen](https://github.com/siddharthvaddem/openscreen)** `TypeScript` ⭐ 18,205
   开源屏幕录制 Demo 制作工具，**无水印、无订阅、可商用**，定位为 Screen Studio 的免费替代品。对 AI 产品 Demo 制作者来说非常实用 🎬

5. **[Yeachan-Heo/oh-my-codex](https://github.com/Yeachan-Heo/oh-my-codex)** `TypeScript` ⭐ 14,139
   为 OpenAI Codex/Agent 添加 **hooks、agent teams、HUD** 等增强功能，打造多 Agent 协作开发环境，Codex 不再孤单 🤖

6. **[google-research/timesfm](https://github.com/google-research/timesfm)** `Python` ⭐ 14,124
   Google Research 出品的**时间序列基础模型**，预训练用于时序预测，覆盖金融、运维、气象等多个场景，小模型大能力 📈

3. **李飞飞 CS231n 开课第 11 年**

4. **Claude Pro 使用额度引争议**
   Reddit 用户抱怨 [Claude Pro 的使用限额](https://www.reddit.com/r/artificial/comments/1sbfwrr/so_what_exactly_is_going_on_with_the_claude_usage/)越来越严格，仅两个重型 Prompt 就消耗了 **73%** 的月度额度。随着模型能力增强，如何在定价与用量间平衡成为 Anthropic 的棘手问题 💰

5. **AI 安全在 Prod 中裸奔**
   Reddit 深度讨论指出企业平均使用 **300+ 未授权 AI 应用**，Prompt Injection 在生产环境中大面积出现。OWASP 和 MITRE ATLAS 虽已有框架，但能落地执行的 AI 安全团队寥寥无几 🛡️

6. **Mamba-3 日志异常检测达 F1=0.9975**
   Reddit 用户分享基于 **Mamba-3/SSM** 架构的日志异常检测项目，在 HDFS 基准上达到 **0.9975 F1**，仅 4.9M 参数，推理延迟低于 2ms。关键突破在于从 BPE 分词转向模板化分词，这是"停止把日志当自然语言处理"的绝佳案例 🎯
