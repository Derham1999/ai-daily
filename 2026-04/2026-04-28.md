### 产品与功能更新

1. **Sam Altman 亲自推荐：Codex $20 计划是笔好买卖**
   OpenAI CEO Sam Altman 在 X 上公开表态：Codex 的 **$20 月付计划** 很划算。这个价格对应 Pro 级订阅，含代码补全、对话问答和 agent 任务执行。Altman 的表态发生在 Codex 改按 token 计费讨论升温的节点，或是在为新的计费模式做预期管理。[来源链接](https://x.com/sama/status/2048913887614115857)

2. **GitHub Copilot 6 月起改按用量计费，重度用户账单变数加大**
   GitHub 宣布从 **6 月 1 日起取消"高级请求次数"限额**，换成按 token 消耗的 AI 积分制。订阅价不变，但重度用户的账单将取决于任务复杂度——简单问答和跑几小时的 agent 任务 token 消耗可能差几个数量级。企业用户有三个月过渡缓冲，5 月初上线预览账单功能。[来源链接](https://x.com/dotey/status/2048849524739977672)

3. **Google Gemma 4 单 GPU 高并发服务演示：10+ 会话无瓶颈**
   Google AI Developers 展示了 Gemma 4 **26B A4B 模型**在单 GPU 上的高并发服务能力：10+ 会话同时运行，系统自动路由、加速、处理，并通过实时仪表盘展示活跃 slot、上下文大小和生成速度。这个 demo 针对 vLLM 等 LLM 服务引擎的效率痛点，强调单卡也能跑复杂任务（如 SVG 生成）。[来源链接](https://x.com/googleaidevs/status/2048824378335641617)

4. **英国 Ineffable Intelligence 获 $1.1B 种子轮，目标超智能体**
   Ineffable Intelligence 宣布获得 **11 亿美元种子轮融资**，这是有史以来英国最大种子轮、欧盟最大种子轮。公司由 DeepMind AlphaGo 核心成员 David Silver 领导，目标是"首次接触超智能"。Emad Mostaque（Stability AI 创始人）评价：团队在自学习模型上已有突破性进展。[来源链接](https://x.com/EMostaque/status/2048790350001533413)

5. **Microsoft Research Forum 5 月 14 日发布 MSR AI Frontiers 新内容**
   Microsoft Research 宣布将于 **5 月 14 日**在 Research Forum 上发布 MSR AI Frontiers 的最新内容，包括 Agentic GitHub Workflows、实时 agent 验证、能量微调等方向的新成果。[来源链接](https://x.com/MSFTResearch/status/2048889966189556059)

### 行业展望与社会影响

1. **OpenAI 与微软重签合作协议：Azure 独占松绑，IP 授权转非独家**
   核心变化一句话概括：**OpenAI 不再被绑在 Azure 上了**。根据新协议，微软仍是主要云合作伙伴，产品优先在 Azure 上线，但如果微软无法或不支持某些能力，OpenAI 可以部署到任何云平台。IP 授权从独家变成 **非独家**，延续到 2032 年。微软不再向 OpenAI 支付分成，而 OpenAI 向微软的分成持续到 2030 年。这个调整本质上是在为 IPO 理顺架构。[来源链接](https://x.com/dotey/status/2048870296531128362)

2. **北京阻止 Meta 收购 AI 创业公司 Manus，出口管制穿透新加坡换壳**
   Meta 试图收购 Manus（AI Agent 创业公司），但 Manus 从中国起家后迁到新加坡。报道称北京 1 月已开始调查其是否违反出口管制，3 月约谈两位联合创始人并限制离境。这被视为 **"Singapore-washing"失效** 的典型案例：公司名义上离岸，但创始人、团队和技术来源仍与中国紧密相连。这次事件释放的信号很明确——AI Agent、核心团队、数据、技术栈，只要与中国境内研发有强关联，即便主体搬到新加坡，也可能被穿透审查。[来源链接](https://newshacker.me/story?id=47925425)

3. **"西方忘了怎么造东西，现在也快忘了怎么写代码"：深度分析引发热议**
   一篇深度文章指出：国防工业花了 **30 年时间**把生产能力"优化"掉，当危机突然要求恢复时，发现"知道怎么造它的人十年前就退休了"。文章警告软件行业正在重演同样剧本：初级工程师跳过调试、跳过犯错过程，就无法建立隐性知识。等我这一代工程师退休时，这些知识不会转移给 AI——它只会消失。这是代码行业的 Fogbank。[来源链接](https://x.com/dotey/status/2048865559027093516)

4. **GitHub 再次故障：PR/Issues 静默错返，迁移 Gitea/Forgejo 讨论升温**
   这次 GitHub 故障主要影响 PR、Issues、Projects 和 Actions。用户看到的是空列表、失败提示或反复刷新后才偶尔恢复的页面。更危险的是：系统会**静默显示没有 open pull requests**，但实际仓库里明明还有很多。很多人借机讨论如何把仓库、CI/CD、Issue tracker 迁出 GitHub，转向 Gitea、Forgejo、GitLab 等自托管方案。[来源链接](https://newshacker.me/story?id=47924775)

### 前沿研究

1. **DAIR.AI 发布系统性研究：AI Agent 如何花钱？**
   这篇论文首次系统研究了 AI Agent 在编程任务上的 token 成本分布。团队在 SWE-bench Verified 上跑了 8 个前沿 LLM，发现大多数团队把 agent token 成本当"四舍五入"，但数据说这不应该被忽略。论文强调：随着自动化程度提高，agent 开销可能快速放大。[来源链接](https://x.com/omarsar0/status/2048802738247450784)

2. **SecureVibeBench：首个聚焦 AI Agent 安全编码的基准**
   论文提出 SecureVibeBench，一个 **105 个 C/C++ 安全编码任务** 的基准。任务源自 OSS-Fuzz 的 41 个项目，要求多文件编辑、覆盖真实漏洞场景。评测结果显示：即使表现最好的 agent，也只有 **23.8%** 能同时产出正确且安全的代码。[来源链接](https://arxiv.org/abs/2509.22097)

3. **AgentBound：首个 MCP 服务器访问控制框架**
   论文指出：数千个 MCP 服务器以无限制权限访问主机系统，攻击面极广。AgentBound 提出声明式策略机制 + 策略执行引擎，在不修改 MCP 服务器的情况下遏制恶意行为。团队构建了最热门 296 个 MCP 服务器的数据集，策略可从源码自动生成，准确率 **80.9%**。[来源链接](https://arxiv.org/abs/2510.21236)

4. **Self-Jailbreak：大推理模型的新型安全失败模式**
   论文发现一种新型失败模式：模型起初能识别有害意图，但在后续推理步骤中**覆盖这个判断**，最终生成不安全输出。这说明安全失败主要源于推理步骤，而非模型不知道什么是错的。论文提出 Chain-of-Guardrail (CoG)，通过 step-level 干预来缓解问题。[来源链接](https://arxiv.org/abs/2510.21285)

### 社媒观点

1. **"AI should elevate your thinking, not replace it."**
   elvis 指出：当前 LLM 并没有开箱即用的能力来真正辅助思考。解决方案是构建自己的 agent harness（检索、验证、记忆、多智能体架构、技能等）。每个输出都应该被复用——让它为你工作，并在下一次会话中变得更好。**关键心态：用 AI 构建你想要的 AI。拥有自己的 harness。**[来源链接](https://x.com/omarsar0/status/2048845158154661961)

2. **Cursor 3 用户反馈整理：不只是"更炫的 IDE"，要可靠的 AI 开发工作台**
   整理自 431 条回复，用户最强烈的诉求是把 Agent、IDE、Git、浏览器、终端、模型选择和团队协作变成**一条稳定的工作流**，而不是在多个模式之间来回切换。核心需求：Agent Window 不能牺牲 IDE 基本能力；多 Agent 并行需要看板式进度视图；Worktree/Git 工作流需要产品级打磨。[来源链接](https://x.com/dotey/status/2048839926679056613)

3. **OpenClaw 4.1x 后 GPT 支持大升级，但用户吐槽："讲话总'卧槽'"**
   用户反馈 OpenClaw 对 GPT 的支持上了大台阶，"只说不干"的毛病基本没了。但修改 soul 文件后，AI 讲话风格变得过于口语化，"像刚走向社会的二愣子"。用户同时吐槽发版频繁，质疑测试环节是否被压缩。[来源链接](https://x.com/frxiaobei/status/2048805385537306677)

4. **GPT-Image 2 生成能力被热议：一张图建起整座史前博物馆**
   用户分享：让 GPT-Image 2 生成一个史前捕食者，它不仅画出了动物，还**围绕它搭建了整个博物馆场景**。Greg Brockman 转发并评价："不只是图片，像在发现历史。"[来源链接](https://x.com/gdb/status/2048796696755212660)
