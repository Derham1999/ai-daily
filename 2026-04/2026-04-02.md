### 产品与功能更新

1. **Claude Code 终端全屏渲染模式上线，代号 NO_FLICKER 解决闪屏顽疾** 🖥️
   Anthropic 为 Claude Code 推出全新终端渲染模式，用户只需设置 `CLAUDE_CODE_NO_FLINKER=1` 环境变量即可启用。该模式解决了长对话场景下终端反复闪屏、跳屏的痛点，尤其在 VS Code 内置终端和 tmux 中效果显著。底层通过优化 ANSI 转义码渲染逻辑实现平滑刷新。[来源](https://x.com/dotey/status/2039447849675469060)

2. **NVIDIA NemoClaw 开源两周，GTC 发布后生态快速成型** 🦞
   NVIDIA 在 GTC 大会发布的开源参考栈 NemoClaw 已满两周，官方陆续放出 DGX Spark 安装教程，展示 NemoClaw + OpenShell + OpenClaw 的协作架构。社区反馈安装体验流畅，首个 @clawbuilders 线下 meetup 已在 rootly 举办，开发者开始分享实际项目案例。[来源](https://x.com/NVIDIAAIDev/status/2039477913167262132)

3. **NVIDIA TAO 多模态微调大幅简化，ExecuTorch 本地语音 Agent 落地** 🔧
   NVIDIA 最新版 TAO 工具让多模态 AI 微调流程大幅简化，降低开发门槛。同时 PyTorch ExecuTorch 新博客展示 LM Studio 如何在本地运行语音 Agent，配合 NVIDIA 硬件实现低延迟推理，移动端 AI Agent 部署再进一步。[来源](https://x.com/NVIDIAAIDev/status/2039447715063468150)

4. **M5 Pro/iOS 跑 100B+ MoE 模型：TurboQuant KV 压缩 + SSD Expert Streaming** 📱
   开发者尝试在 Apple Silicon 上本地运行超大 MoE 模型，将 TurboQuant 论文的 V3 Lloyd-Max codebooks 移植到 native C++，并将反量化融合进 Metal shaders。结合 SSD Expert Streaming 技术，实现在移动设备上推理 100B+ 参数模型。社区质疑缺乏真实 benchmark 验证。[来源](https://newshacker.me/story?id=47604354)

5. **Cloudflare 推 EmDash：TypeScript 版 WordPress 继任者，插件安全成焦点** 🌐
   Cloudflare 发布开源 CMS 项目 EmDash，定位为 WordPress 的精神继任者。基于 TypeScript + Astro 构建，运行在 Cloudflare Workers + D1 边缘架构上。HN 评分高达 265 分，但社区争议集中在是否只能跑在 Cloudflare 自家云上、插件生态安全性等问题。[来源](https://newshacker.me/story?id=47602832)

### 行业展望与社会影响

1. **OpenAI "墓地"清单曝光：泡沫、烂尾与变现困局** 📉
   Forbes 发布 OpenAI Graveyard 清单，列举了 OpenAI 过去高调宣传但最终取消、搁置或迟迟未落地的合作与产品。HN 评分 138 分引发广泛讨论——项目一堆烂尾，估值却持续攀升。这一清单折射出 AI 行业普遍存在的"过度承诺"现象，也引发对 AI 公司估值合理性的反思。[来源](https://newshacker.me/story?id=47602565)

2. **OpenClaw Arena 300+ 场实测：StepFun 3.5 Flash 性价比第一** 💸
   OpenClaw Arena 评测平台通过 300+ 场真实任务对战，测出 StepFun 3.5 Flash 在性价比维度排名第一，性能排名第五。该平台让用户提交任务、盲选 2-5 个模型，由 judge agent 在隔离环境评判，为模型选型提供了真实场景下的参考依据。[来源](https://newshacker.me/story?id=47602879)

3. **从计算尺到 AI：工程知识正在失传的警示** 📐
   一篇引发 61 分热议的文章回顾了从计算尺时代到 AI 时代的工程实践演变，指出很多核心能力并未完整记录在文档中，而是依赖长期训练和经验手感。评论区将话题延伸至 NASA Saturn V 与 SLS 的对比，引发了"AI 辅助下工程师是否在丢掉根本技能"的深层讨论。[来源](https://newshacker.me/story?id=47602500)

### 前沿研究

1. **多智能体协作自动化研究实证：Agent 团队做科研效果如何？** 🤖
   [论文](https://arxiv.org/abs/2603.29632)对多 Agent 协作进行自动化研究进行了实证分析，探讨多个 AI Agent 组队完成科研任务的效果与局限，为 AI 辅助科研提供系统化评估。

2. **多模态大语言模型对抗性提示注入攻击** 🛡️
   [论文](https://arxiv.org/abs/2603.29418)系统研究了针对多模态 LLM 的对抗性提示注入攻击，揭示了视觉+文本多模态场景下的新型安全威胁，与 Claude Code 源码中发现的"工具输出免疫系统"设计形成呼应。

3. **PAR²-RAG：多跳问答的主动检索与推理** 🔍
   [论文](https://arxiv.org/abs/2603.29085)提出 Planned Active Retrieval and Reasoning 方法，通过规划式主动检索提升多跳问答性能，为 RAG 系统的复杂推理能力提供新思路。

### 社媒观点

1. **Claude Code 源码泄露后，HKUDS 推出 OpenHarness 轻量替代** 🪶
   香港大学数据科学实验室推出 OpenHarness，仅用 **2.3% 的代码量**（11,733 行 vs Claude Code 的 512,664 行）实现 80% 核心功能，代码库轻量 44 倍。为研究者理解和实验 AI Agent 架构提供了极简入口。[来源](https://x.com/shao__meng/status/2039509008147710200)

2. **EdgeClaw 2.0 融入四级记忆架构，结构化长时记忆成趋势** 🧠
   OpenBMB 发布 EdgeClaw 2.0，借鉴 Claude Code 的多级记忆设计实现四级架构：L0 原始对话 → L1 记忆片段 → L2 项目/时间记忆 → L3 聚合知识。结构化、可演进的长时记忆系统正在成为 AI Agent 的标配。[来源](https://x.com/shao__meng/status/2039504271411151016)

3. **智谱发布 GLM-skills：模型能力的 Skill-as-Code 框架** ⚡
   智谱 AI 官方发布 GLM-skills 框架，将 GLM 系列模型的原子能力封装为标准化、可插拔的 Skill 模块。最新发布的 GLM-5V-Turbo 对应多模态 Skills，体现"解耦与聚合"的设计哲学，支持各类 Agent 架构直接调用。[来源](https://x.com/shao__meng/status/2039500261119709345)

4. **Google DeepMind 新研究：开放网络本身才是 AI Agent 最大威胁** 🕸️
   Google DeepMind 发表论文指出，AI Agent 面临的最大威胁不是更强的攻击者，而是开放 Web 本身——恶意网页内容可通过提示注入操控 Agent 行为。该研究建立了首个系统性框架来理解开放 Web 对 Agent 的安全威胁。[来源](https://x.com/omarsar0/status/2039383554510217707)

5. **Claude Code 源码解读热潮下的冷思考：能跑出规模的才是真本事** 💭
   业内人士指出，Claude Code 源码泄露后的各种解读大多是 AI 在复述 AI，真正能认真拆解的寥寥无几。历史上 Google、百度代码流出后也未能催生出同等规模的竞争者，代码可复制，但工程能力和规模效应不可复制。[来源](https://x.com/frxiaobei/status/2039380180754735305)

6. **Zerobox：AI Agent 安全沙箱工具，限制文件与网络访问** 🔒
   Zerobox 是一个 CLI 沙箱工具，可在执行任意命令时限制文件和网络访问，并通过网络代理注入密钥。底层依赖 OpenAI Codex 的 Rust crates，结合 macOS seatbelt 和 Linux bubblewrap 实现跨平台沙箱隔离。HN 评分 40 分。[来源](https://newshacker.me/story?id=47574871)
