3. **飞书 aily 全面升级，推出企业级安全"龙虾"（Claude）集成方案。**
   飞书将内部 AI 助手 aily 升级为更安全、与飞书深度集成的企业 Agent，**一键配置**后生成独立联系人助手，可读取飞书所有信息、执行任务、调用外部工具——主打安全合规，解决 to B 企业自部署 Claude 的顾虑。[来源](https://x.com/op7418/status/2034584589986042298) 🐦 企业版龙虾来了，大厂效率工具迎来新一轮 AI 升级战。

4. **MolmoPoint 开源视觉定位模型：8B 参数，直接从视觉特征选坐标，绕过数字转换。**
   基于 Qwen3 8B，MolmoPoint **直接从视觉特征中选择目标位置**，避免"视觉→坐标数字→文本"的低效路径。开源了通用视觉、GUI 专用、视频专用三个版本，**同尺寸模型视觉能力最佳**。[HuggingFace](https://huggingface.co/collections/allenai/molmopoint) 🌟

5. **Chandra OCR 2：开源 OCR 模型，4B 参数，评分超越小红书 dots.ocr-1.5。**
   以更小参数量取得更高 OCR 评分，在多项基准上直接击败小红书开源的 dots.ocr-1.5。[模型地址](https://huggingface.co/datalab-to/chandra-ocr-2) 📝 开源 OCR 赛道竞争白热化，参数效率越来越成为核心竞争力。

6. **NVIDIA NemoClaw：为 OpenClaw Agent 打造的安全沙箱插件，策略管控网络/文件/推理全链路。**
   在 OpenClaw 成为自主 AI "操作系统"的背景下，NemoClaw 引入 NVIDIA OpenShell 运行时，用声明式策略管控 Agent 的一切外部交互，实现 **24×7 自主运行**的同时防止越权访问。[来源](https://x.com/shao__meng/status/2034604533347971460) 🛡️

### 前沿研究

1. **SPEED-Bench：推测解码首个统一多样化评测基准，覆盖多模型/任务/硬件场景。**
   NVIDIA 联合 HuggingFace 提出，解决推测解码（Speculative Decoding）评测碎片化问题，有望成为该方向的标准测试平台。**推测解码**通过小模型草稿+大模型验证提升推理速度，统一基准的出现将加速该技术的落地。[HuggingFace Blog](https://huggingface.co/blog/nvidia/speed-bench)

2. **VLM 越狱防御新思路：从越狱相关表征空间理解并抵御攻击。**
   *Understanding and Defending VLM Jailbreaks via Jailbreak-Related Representations* 分析视觉大模型内部的越狱响应机制，提出基于**内部表征**的防御方法，有别于传统输入过滤方案。🛡️ 随着多模态模型加速部署，VLM 安全正成为研究热点。[arXiv](https://arxiv.org/abs/2603.17372)

3. **Just-in-Time：扩散 Transformer 无需训练的空间加速，动态跳过冗余计算。**
   针对 DiT 推理慢的痛点，提出动态跳过冗余空间计算的策略，**无需额外训练**即可显著加速，对图像生成应用的实时部署有重要价值。[arXiv](https://arxiv.org/abs/2603.10744) ⚡

4. **Versatile Video Editing：无需训练即可统一编辑视频内容、动作与动态的通用框架。**
   现有视频编辑方法通常针对特定任务微调，本文提出同时支持**内容、动作、动态**三类编辑的统一框架，无需重新训练。[arXiv](https://arxiv.org/abs/2603.17989) 🎬

5. **从行车记录仪视频自动判定交通责任：法律 Multi-Agent 推理落地现实场景。**
   结合法律推理与多 Agent 系统，*Interpretable Traffic Responsibility* 从行车记录仪视频中提取可解释的责任判断，在保险理赔和司法辅助场景有直接应用潜力。[arXiv](https://arxiv.org/abs/2603.17930) 🚗

6. **MiroThinker H1：少推理步骤+强验证，43% 更少交互轮次实现 17% 性能提升。**
   Reddit 技术拆解：通过**更短推理链 + 更严格验证步骤**代替长循环，比上一代减少 43% 交互轮次、性能提升 17%。对解决 Agent 陷入无效 tool call 循环的工程问题有直接参考价值。[Reddit](https://www.reddit.com/r/MachineLearning/comments/1rxz4xk/d_breaking_down_mirothinker_h1s_verification/)

### 社媒观点

1. **Agent 指令设计的核心悖论："像人思考"与"像人行动"是两条截然不同的路。**
   Orange AI 转发的洞察指出：当前提示词设计普遍追求"像人一样"，但**思维方式人性化**与**行动方式人性化**是两套完全不同的系统，混淆两者会导致 Agent 行为大幅偏差。[来源](https://x.com/oran_ge/status/2034623129902043250) 🤔 这是 Agent 工程师值得细品的底层设计原则。

2. **个人 AI 记忆系统落地指南：三个纯文本 MD 文件，让 AI 永久"认识"你。**
   USER.md（身份画像）+ MEMORY.md（长期洞察）+ CONTEXT.md（当前任务），零数据库、零向量库。核心洞察：**行业在追"聪明"，用户需要的是"熟悉"**——AI 永远从零上下文开始，才是真正的体验瓶颈。[来源](https://x.com/shao__meng/status/2034595778480763049) 📝

3. **AI 工具最大鸿沟：为开发者构建，普通用户根本无法真正用起来。**
   一位 AI Agent 开发者直言：餐厅老板、小商家等真正需要 AI 自动化的群体，根本无法独立部署和调试，**AI 效率革命还没有真正触达大众**，这才是下一个十亿级市场的真正机会。[Reddit](https://www.reddit.com/r/artificial/comments/1rxuu18/most_ai_tools_are_built_for_developers_heres_what/) 🌐

4. **用 Claude Code 读财报、学投资：按需构建个人知识库的 AI 协作新姿势。**
   吕立青分享实践：每遇新金融概念就让 Claude Code 写一份解析放进 knowledge base，**干中学效果极佳**。这种"边用边积累"的方式，正在成为知识工作者使用 AI 的新范式。[来源](https://x.com/jimmy_lv/status/2034531068590014465) 💡
